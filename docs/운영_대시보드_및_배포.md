# 운영 대시보드 실행 및 배포

서비스를 직접 운영하려면 **웹 대시보드**를 로컬에서 실행하거나 배포해 사용하면 됩니다.

---

## 1. 단계별 작업 (먼저 완료할 것)

전체 준비 순서는 **`docs/단계별_작업_체크리스트.md`** 를 보고 1~6단계까지 진행한 뒤, 7단계에서 아래 대시보드를 실행/배포하세요.

---

## 2. 로컬에서 대시보드 실행

### 2-1. 환경 변수

`web` 폴더에 `.env.local` 파일을 만듭니다.

```env
SUPABASE_URL=https://rchflauwrrlfefxzbtks.supabase.co
SUPABASE_SERVICE_ROLE_KEY=여기에_서비스_롤_키
```

- **SUPABASE_URL**: Supabase 대시보드 → **Settings → API** → Project URL  
- **SUPABASE_SERVICE_ROLE_KEY**: 같은 화면에서 **service_role** (secret) 복사  
  → 이 키는 서버에서만 사용하고, 브라우저·공개 저장소에 올리지 마세요.

### 2-2. 실행

```bash
cd "c:\Users\PC\Desktop\영업 메일링\web"
npm install
npm run dev
```

브라우저에서 **http://localhost:3000** 으로 접속합니다.

### 2-3. 대시보드에서 할 수 있는 것

- **브랜드 목록** 조회 (상태별 개수 표시)
- **AI 제안 생성** 버튼 → `수집완료` 브랜드에 대해 제안 생성
- **메일 발송** 버튼 → `발송대기` 브랜드에게 메일 발송
- **새로고침** → 목록 다시 불러오기

발송할 브랜드는 Supabase **Table Editor**에서 해당 행의 `status` 를 **발송대기**로 바꾼 뒤 메일 발송을 실행하면 됩니다.

## 3. 배포해서 URL 로 서비스 운영하기 (Vercel)

인터넷에서 접속 가능한 URL을 쓰려면 Vercel에 배포하면 됩니다.

### 3-1. 준비

- [Vercel](https://vercel.com) 계정
- 이 프로젝트를 Git 저장소에 올려두거나, Vercel이 해당 폴더에 접근 가능해야 함

### 3-2. Vercel에서 프로젝트 생성

1. [Vercel](https://vercel.com) 로그인 후 **Add New… → Project**
2. 저장소 선택(또는 **Import Git Repository** 로 이 프로젝트 연결)
3. **Root Directory** 를 `web` 으로 지정 (한 단계만 들어가서 `web` 선택)
4. **Environment Variables** 에 추가:
   - 이름: `SUPABASE_URL`  
     값: `https://rchflauwrrlfefxzbtks.supabase.co`
   - 이름: `SUPABASE_SERVICE_ROLE_KEY`  
     값: (Supabase 서비스 롤 키)
5. **Deploy** 클릭

### 3-3. 배포 후 URL

배포가 끝나면 다음처럼 URL이 발급됩니다.

- 예: `https://영업메일링-xxx.vercel.app`
- 또는 커스텀 도메인 연결 가능 (Vercel 설정에서 추가)

이 URL로 접속하면 **서비스를 직접 운영**할 수 있습니다.  
같은 화면에서 브랜드 확인 → AI 제안 생성 → 메일 발송까지 진행하면 됩니다.

### 3-4. (선택) 로컬에서 Vercel CLI로 배포

```bash
npm i -g vercel
cd "c:\Users\PC\Desktop\영업 메일링\web"
vercel
```

프롬프트에서 Root를 `./` 로 두고, `web` 이 현재 폴더라면 그대로 배포됩니다.  
다른 루트에서 배포할 경우 **Root Directory** 를 `web` 으로 지정하세요.
