# 수집 실행 – 단계별 설정 가이드 (접속 주소 포함)

Vercel 사이트에서 **어디서 접속해도** "수집 실행" 버튼이 동작하도록 하려면, 크롤러 전용 서비스를 한 번 배포하고 Vercel과 연결해야 합니다.

---

## 1단계: Railway 가입 및 프로젝트 만들기

| 항목 | 내용 |
|------|------|
| **접속 주소** | https://railway.app |
| **할 일** | 브라우저에서 접속 후 **Login** → GitHub로 로그인 |

로그인 후:

1. **New Project** 버튼 클릭  
2. **Deploy from GitHub repo** 선택  
3. **Configure GitHub App** 이 나오면 **설정**해서 이 프로젝트 저장소(영업 메일링)에 Railway 접근 허용  
4. 저장소 목록에서 **해당 저장소 선택** (예: `sabuja12344-art/mail-sender`) 후 **Deploy Now**

---

## 2단계: Railway에서 Dockerfile 지정

배포가 시작된 뒤, **해당 서비스(카드)** 를 클릭해서 들어갑니다.

| 메뉴 위치 | 할 일 |
|-----------|--------|
| **Settings** 탭 | 왼쪽 또는 상단 **Settings** 클릭 |
| **Build** 섹션 | 아래 항목 설정 |
| **Dockerfile Path** | `Dockerfile.crawler` 입력 (기본값 비우고 이 값만) |
| **Root Directory** | 비워 두기 (또는 `.`) |
| **Start Command** | 비워 두기 |

**Save** 또는 **Deploy** 버튼이 있으면 눌러서 다시 빌드됩니다.

---

## 3단계: Railway 환경 변수 넣기

같은 서비스 안에서:

| 메뉴 위치 | 할 일 |
|-----------|--------|
| **Variables** 탭 | **Variables** 클릭 |
| **+ New Variable** 또는 **RAW Editor** | 아래 변수 추가 |

| 변수 이름 | 값 (예시) | 비고 |
|-----------|-----------|------|
| `SUPABASE_URL` | `https://rchflauwrrlfefxzbtks.supabase.co` | Supabase 프로젝트 URL |
| `SUPABASE_SERVICE_ROLE_KEY` | `eyJhbGc...` (긴 문자열) | Supabase service_role 키 |
| `CRAWLER_SERVICE_SECRET` | 예: `my-secret-123` | 아무 비밀 문자열 (나중에 Vercel에도 동일하게 입력) |

- **Supabase URL·키 확인:**  
  - 접속: https://supabase.com/dashboard  
  - 프로젝트 선택 → **Settings** (왼쪽 아래) → **API**  
  - **Project URL** → `SUPABASE_URL`  
  - **Project API keys** 에서 **service_role** (비공개) → `SUPABASE_SERVICE_ROLE_KEY`

저장 후 필요하면 **Redeploy** 한 번 실행합니다.

---

## 4단계: Railway 공개 URL 받기

| 메뉴 위치 | 할 일 |
|-----------|--------|
| **Settings** 탭 | **Settings** 클릭 |
| **Networking** 섹션 | **Generate Domain** 버튼 클릭 |

생성된 주소가 예: `https://영업메일링-production-xxxx.up.railway.app` 형태로 나옵니다.

- 이 **전체 주소를 복사**합니다. (마지막에 `/` 없이, `https://...` 까지만)  
- 이 주소가 **크롤러 서비스 접속 주소**입니다.  
- (선택) 브라우저에서 `https://생성된주소/health` 로 접속해 `{"ok":true,"service":"crawler"}` 가 나오면 정상입니다.

---

## 5단계: Vercel 환경 변수 넣기

| 항목 | 내용 |
|------|------|
| **접속 주소** | https://vercel.com/dashboard |
| **할 일** | 로그인 후 **영업 메일링(메일 발송)** 프로젝트 클릭 |

프로젝트 안에서:

1. 상단 메뉴에서 **Settings** 클릭  
2. 왼쪽에서 **Environment Variables** 클릭  
3. **Add New** (또는 **Key**, **Value** 입력란) 에서 아래 두 개 추가:

| Name (키) | Value (값) | 적용 환경 |
|-----------|------------|-----------|
| `CRAWLER_SERVICE_URL` | 4단계에서 복사한 Railway 주소 (예: `https://영업메일링-production-xxxx.up.railway.app`) | Production, Preview, Development 원하는 것 체크 |
| `CRAWLER_SERVICE_SECRET` | 3단계에서 Railway에 넣은 것과 **완전히 동일** (예: `my-secret-123`) | 위와 동일 |

- **Save** 후, 이미 배포된 사이트가 있다면 **한 번 재배포**하는 것이 좋습니다.  
  - **Deployments** 탭 → 최신 배포 오른쪽 **⋮** → **Redeploy** 로 재배포 가능.

---

## 6단계: 동작 확인

| 항목 | 내용 |
|------|------|
| **접속 주소** | 본인 Vercel 사이트 주소 (예: `https://메일링프로젝트.vercel.app`) |
| **할 일** | **다른 기기나 다른 네트워크**에서도 접속해 봅니다. |

1. 로그인/대시보드 들어가기  
2. 키워드 입력 (예: `이불`)  
3. **수집 실행** 버튼 클릭  
4. "N개가 정상 수집되었습니다" 또는 크롤러 완료 메시지가 나오면 성공입니다.

---

## 요약 – 접속 주소 정리

| 용도 | 접속 주소 |
|------|-----------|
| Railway (크롤러 서비스 배포·설정) | https://railway.app |
| Railway 크롤러 상태 확인 (선택) | `https://[4단계에서 받은 도메인]/health` |
| Supabase (URL·키 확인) | https://supabase.com/dashboard → 프로젝트 → Settings → API |
| Vercel (환경 변수·재배포) | https://vercel.com/dashboard → 프로젝트 → Settings → Environment Variables |
| 실제 사용하는 사이트 (수집 실행 테스트) | 본인 Vercel 사이트 URL (예: `https://xxx.vercel.app`) |

## 문제가 생겼을 때

- **"크롤러 실행 실패" / "크롤러 서비스 오류"**  
  - Vercel의 `CRAWLER_SERVICE_URL` 이 Railway에서 받은 주소와 **완전히 동일**한지 확인 (끝에 `/` 없이).  
  - Railway 서비스가 **실행 중**(초록색)인지 확인.  
- **Railway 빌드 실패**  
  - **Dockerfile Path** 가 정확히 `Dockerfile.crawler` 인지 확인.  
- **수집은 되는데 DB에 안 쌓임**  
  - Railway **Variables** 의 `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` 가 Supabase **API** 화면 값과 같은지 확인.

이 가이드대로 하면 **접속하는 곳과 관계없이** Vercel 사이트에서 수집 실행을 사용할 수 있습니다.
