# 수집 실행 – 비용 없이 설정하기 (GitHub Actions)

Railway 크레딧 없이 **완전 무료**로 "수집 실행"을 쓰려면 **GitHub Actions**를 사용하면 됩니다.  
버튼을 누르면 GitHub에서 크롤러가 실행되고, 2~3분 뒤 새로고침하면 수집 결과가 반영됩니다.

---

## 1단계: GitHub 저장소 시크릿 넣기

| 접속 주소 | https://github.com |
|-----------|--------------------|
| 이동 경로 | 본인 계정 → **영업 메일링 저장소** (예: `sabuja12344-art/mail-sender`) 클릭 |

1. 저장소 페이지에서 **Settings** 탭 클릭  
2. 왼쪽 메뉴 **Secrets and variables** → **Actions** 클릭  
3. **New repository secret** 클릭해서 아래 두 개 추가:

| Name (이름) | Secret (값) |
|-------------|-------------|
| `SUPABASE_URL` | Supabase 프로젝트 URL (예: `https://xxxx.supabase.co`) |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase **service_role** 키 (Settings → API에서 복사) |

- Supabase URL·키 확인: https://supabase.com/dashboard → 프로젝트 선택 → **Settings** → **API**

---

## 2단계: GitHub 토큰 만들기 (워크플로 트리거용)

| 접속 주소 | https://github.com/settings/tokens |
|-----------|-------------------------------------|
| 또는 경로 | GitHub 우측 상단 프로필 클릭 → **Settings** → 왼쪽 맨 아래 **Developer settings** → **Personal access tokens** → **Tokens (classic)** |

1. **Generate new token** → **Generate new token (classic)** 선택  
2. **Note**: 예) `영업메일링 크롤러 트리거`  
3. **Expiration**: 원하는 기간 (예: 90 days 또는 No expiration)  
4. **Select scopes**에서 **repo** 체크 (또는 최소한 해당 저장소 권한)  
5. **Generate token** 클릭 후 나온 토큰을 **한 번만** 복사해 둡니다 (다시 안 보입니다).

---

## 3단계: Vercel 환경 변수 넣기

| 접속 주소 | https://vercel.com/dashboard |
|-----------|------------------------------|
| 이동 경로 | **영업 메일링 프로젝트** 클릭 → **Settings** → **Environment Variables** |

**Add New**로 아래 두 개 추가:

| Name (키) | Value (값) |
|-----------|------------|
| `GITHUB_ACTIONS_CRAWLER_REPO` | GitHub 저장소 주소 (예: `sabuja12344-art/mail-sender`) — `owner/repo` 형식, 슬래시 포함 |
| `GITHUB_ACTIONS_CRAWLER_TOKEN` | 2단계에서 만든 Personal access token (토큰 전체 문자열) |
| (선택) `GITHUB_ACTIONS_CRAWLER_REF` | 기본 브랜치가 `main`이 아니면 `master` 등 입력. 비우면 `main` 사용 |

- **중요:** `CRAWLER_SERVICE_URL`은 **설정하지 않습니다**. 비워 두면 GitHub Actions 방식이 사용됩니다.  
- 저장 후 **Deployments**에서 최신 배포 **Redeploy** 한 번 해 주세요.

---

## 4단계: 워크플로 파일이 저장소에 있는지 확인

이 가이드대로 진행했다면, 저장소에 아래 파일이 있어야 합니다.

- `.github/workflows/run-crawler.yml`

없다면 이 파일을 추가한 뒤 **main** 브랜치에 푸시합니다.  
(이미 프로젝트에 포함되어 있으면 추가 작업 없음.)

---

## 5단계: 사용 방법

| 접속 주소 | 본인 Vercel 사이트 (예: `https://xxx.vercel.app`) |

1. 대시보드에서 키워드 입력 (예: `이불`)  
2. **수집 실행** 버튼 클릭  
3. "수집이 백그라운드에서 시작되었습니다. 2~3분 후 목록을 새로고침하세요." 메시지 확인  
4. **2~3분 후** 페이지 **새로고침** → 수집된 목록 확인  

---

## 요약 – 접속 주소 정리

| 용도 | 접속 주소 |
|------|-----------|
| 저장소 시크릿 설정 | https://github.com → 해당 저장소 → **Settings** → **Secrets and variables** → **Actions** |
| 토큰 발급 | https://github.com/settings/tokens |
| Supabase URL·키 확인 | https://supabase.com/dashboard → 프로젝트 → **Settings** → **API** |
| Vercel 환경 변수 | https://vercel.com/dashboard → 프로젝트 → **Settings** → **Environment Variables** |
| 실제 사용 (수집 실행) | 본인 Vercel 사이트 URL |

## Railway와 비교

| 구분 | GitHub Actions (무료) | Railway (유료 크레딧) |
|------|------------------------|------------------------|
| 비용 | 무료 (공개 저장소 무제한, 비공개는 월 2000분 등 한도 내) | $5 크레딧 등 사용량만큼 차감 |
| 결과 확인 | 버튼 클릭 후 2~3분 뒤 **새로고침** 필요 | 즉시 결과 메시지 가능 (N건 수집) |
| 설정 | GitHub 시크릿 + Vercel 환경 변수 | Railway 배포 + Vercel 환경 변수 |

**비용 없이 쓰려면** 위 GitHub Actions 방식만 설정하고, `CRAWLER_SERVICE_URL`은 넣지 않으면 됩니다.
