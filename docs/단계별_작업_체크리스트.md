# 영업 메일링 서비스 — 단계별 작업 체크리스트

아래 순서대로 진행하면 서비스를 직접 운영할 수 있습니다.

---

## 1단계: Supabase DB 준비

- [ ] **1-1.** [Supabase 대시보드](https://supabase.com/dashboard) 로그인 후 프로젝트 선택
- [ ] **1-2.** **SQL Editor**에서 `brands` 테이블 생성  
  → `supabase/migrations/20250310000000_create_brands_table.sql` 내용 전체 복사 후 **Run**
- [ ] **1-3.** 같은 SQL Editor에서 `확인완료` 상태 추가  
  → 아래 한 줄 실행:
  ```sql
  ALTER TYPE brand_status ADD VALUE '확인완료';
  ```

---

## 2단계: 환경 변수·시크릿 설정

- [ ] **2-1.** 프로젝트 루트에 `.env` 파일 존재 확인  
  → `SUPABASE_URL`, `SUPABASE_ANON_KEY` 가 있어야 함 (이미 설정됨)
- [ ] **2-2.** Supabase 대시보드 → **Project Settings → Edge Functions → Secrets** 에서 아래 추가:
  - `GEMINI_API_KEY` — [Google AI Studio](https://aistudio.google.com/apikey)에서 발급
  - `RESEND_API_KEY` — [Resend](https://resend.com)에서 발급 (`re_` 로 시작)
  - `RESEND_FROM_EMAIL` — 발신 이메일 (예: `onboarding@resend.dev` 또는 인증한 도메인)
  - (선택) `PROPOSAL_VIEWED_REDIRECT_URL` — 제안서 상세보기 클릭 후 이동할 URL

---

## 3단계: Edge Function 배포

- [ ] **3-1.** 터미널에서 프로젝트 폴더로 이동:
  ```bash
  cd "c:\Users\PC\Desktop\영업 메일링"
  ```
- [ ] **3-2.** Supabase 로그인 및 프로젝트 연결 (한 번만):
  ```bash
  npx supabase login
  npx supabase link --project-ref rchflauwrrlfefxzbtks
  ```
- [ ] **3-3.** Edge Function 3개 배포:
  ```bash
  npx supabase functions deploy generate-proposals
  npx supabase functions deploy send-proposal-emails
  npx supabase functions deploy proposal-viewed
  ```
- [ ] **3-4.** 대시보드 **Edge Functions** 메뉴에서 위 3개 함수가 보이는지 확인

---

## 4단계: 브랜드 수집 (크롤러)

- [ ] **4-1.** Python 환경 준비 (한 번만):
  ```bash
  pip install -r requirements.txt
  playwright install chromium
  ```
- [ ] **4-2.** 키워드로 브랜드 수집:
  ```bash
  python crawler/brand_crawler.py
  ```
  또는 `python crawler/brand_crawler.py 여성의류` 처럼 키워드 지정
- [ ] **4-3.** Supabase **Table Editor → brands** 에서 행이 늘었는지 확인 (`status`: 수집완료)

---

## 5단계: AI 제안서 생성

- [ ] **5-1.** 운영 대시보드에서 **「AI 제안 생성」** 버튼 클릭  
  (또는 직접 호출: 대시보드 **Edge Functions → generate-proposals → Invoke**)
- [ ] **5-2.** `brands` 테이블에서 `status` 가 **제안생성** 으로 바뀌고, `analysis_summary` 에 내용이 채워졌는지 확인

---

## 6단계: 발송대기 설정 후 메일 발송

- [ ] **6-1.** 발송할 브랜드의 `status` 를 **발송대기** 로 수동 변경  
  (Table Editor에서 해당 행의 `status` 셀 클릭 후 **발송대기** 선택)
- [ ] **6-2.** 운영 대시보드에서 **「메일 발송」** 버튼 클릭  
  (또는 **Edge Functions → send-proposal-emails → Invoke**)
- [ ] **6-3.** 발송된 메일 수신 확인 및 메일 내 **「제안서 상세보기」** 링크 클릭 시  
  해당 브랜드 `status` 가 **확인완료** 로 바뀌는지 확인

---

## 7단계: 운영 대시보드 실행 및 배포

- [ ] **7-1.** `web` 폴더에 `.env.local` 생성 후 `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` 입력  
  (자세한 값·위치: **`docs/운영_대시보드_및_배포.md`** 참고)
- [ ] **7-2.** 로컬 실행:
  ```bash
  cd web
  npm install
  npm run dev
  ```
  → 브라우저에서 **http://localhost:3000** 접속
- [ ] **7-3.** (선택) Vercel 배포 후 공개 URL로 서비스 운영  
  → **`docs/운영_대시보드_및_배포.md`** 의 「배포해서 URL 로 서비스 운영하기」 참고
- [ ] **7-4.** 대시보드에서 브랜드 목록 조회, **AI 제안 생성**, **메일 발송** 버튼 동작 확인

---

## 완료 후 일상 운영 흐름

1. **크롤러**로 새 브랜드 수집 → `brands` 에 추가 (수집완료)
2. **운영 대시보드** (localhost:3000 또는 배포 URL)에서 **AI 제안 생성** 클릭 → 제안생성
3. Supabase Table Editor에서 발송할 브랜드만 **발송대기** 로 변경
4. 대시보드에서 **메일 발송** 클릭 → 발송완료
5. 수신자가 메일의 **제안서 상세보기** 클릭 → 확인완료
