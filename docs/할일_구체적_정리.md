# 할 일 — 구체적으로 정리

아래 **번호 순서대로** 진행하세요. 이미 했다면 ✓ 표시하고 다음으로 넘어가면 됩니다.

---

## □ 할 일 1 — Supabase에서 brands 테이블 있는지 확인

1. 브라우저에서 **https://supabase.com/dashboard** 접속 → 로그인 → 프로젝트 선택.
2. 왼쪽 메뉴 **Table Editor** 클릭.
3. 목록에 **brands** 가 있으면 ✓. 없으면 **할 일 2** 로 가서 SQL 실행.

---

## □ 할 일 2 — brands 테이블이 없을 때만: SQL로 테이블 생성

1. Supabase 왼쪽 메뉴 **SQL Editor** 클릭 → **+ New query**.
2. 아래 SQL **전체**를 복사해 에디터에 붙여넣기 → **Run** 클릭.

```sql
CREATE TYPE brand_status AS ENUM (
  '수집완료',
  '제안생성',
  '발송대기',
  '발송완료',
  '회신완료'
);

CREATE TABLE public.brands (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  website_url TEXT,
  email TEXT,
  pixel_installed BOOLEAN NOT NULL DEFAULT false,
  analysis_summary TEXT,
  status brand_status NOT NULL DEFAULT '수집완료',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_brands_status ON public.brands (status);
CREATE INDEX idx_brands_created_at ON public.brands (created_at DESC);

ALTER TABLE public.brands ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow authenticated users full access to brands"
  ON public.brands FOR ALL TO authenticated USING (true) WITH CHECK (true);

CREATE POLICY "Allow service role full access to brands"
  ON public.brands FOR ALL TO service_role USING (true) WITH CHECK (true);

COMMENT ON TABLE public.brands IS '수집된 브랜드 정보와 발송 상태 관리';
COMMENT ON COLUMN public.brands.name IS '브랜드명';
COMMENT ON COLUMN public.brands.website_url IS '웹사이트 주소';
COMMENT ON COLUMN public.brands.email IS '추출된 대표 이메일';
COMMENT ON COLUMN public.brands.pixel_installed IS '픽셀 설치 여부';
COMMENT ON COLUMN public.brands.analysis_summary IS 'AI가 분석한 브랜드 약점/특징';
COMMENT ON COLUMN public.brands.status IS '상태';
COMMENT ON COLUMN public.brands.created_at IS '수집 일자';
```

3. 같은 SQL Editor에서 내용 **전부 지우고**, 아래 **한 줄만** 붙여넣기 → **Run**.

```sql
ALTER TYPE brand_status ADD VALUE '확인완료';
```

## □ 할 일 3 — Supabase Secrets 확인 (API 키 3개)

1. Supabase **Settings(톱니바퀴)** → **Edge Functions** → **Secrets**.
2. 아래 3개가 **전부** 있는지 확인.
   - `GEMINI_API_KEY`
   - `RESEND_API_KEY`
   - `RESEND_FROM_EMAIL`
3. 하나라도 없으면:
   - **Add new secret** 클릭 → Name / Value 입력 → **Save**.
   - Resend 키: **https://resend.com** → 로그인 → **API Keys** → **Create API Key** → 복사 후 Value에 붙여넣기.
   - 발신 이메일 Value: `onboarding@resend.dev` (또는 본인 이메일).

## □ 할 일 4 — Edge Function 3개 배포됐는지 확인

1. Supabase 왼쪽 메뉴 **Edge Functions** 클릭.
2. 목록에 아래 3개가 있는지 확인.
   - `generate-proposals`
   - `send-proposal-emails`
   - `proposal-viewed`
3. 하나라도 없으면: 터미널에서 아래 명령을 **한 줄씩** 실행 (복사 후 엔터).

```
cd "c:\Users\PC\Desktop\영업 메일링"
```

```
npx supabase functions deploy generate-proposals
```

```
npx supabase functions deploy send-proposal-emails
```

```
npx supabase functions deploy proposal-viewed
```

## □ 할 일 5 — web 폴더에 .env.local 만들기

1. **서비스 롤 키** 복사: Supabase **Settings** → **API** → **service_role** 옆 **Reveal** 클릭 → 값 복사.
2. **영업 메일링** 폴더 안 **web** 폴더로 이동.
3. **web** 폴더 안에 새 파일 만들기 → 이름: **`.env.local`** (맨 앞 점 포함).
4. 파일 안에 아래 **두 줄** 넣기. 두 번째 줄의 `여기_서비스_롤_키` 를 1번에서 복사한 값으로 **통째로** 바꾸기.

```
SUPABASE_URL=https://rchflauwrrlfefxzbtks.supabase.co
SUPABASE_SERVICE_ROLE_KEY=여기_서비스_롤_키
```

5. 저장 후 파일 닫기.

## □ 할 일 6 — 대시보드 실행해서 localhost:3000 접속

1. **명령 프롬프트** 또는 **PowerShell** 실행.
2. 아래를 **한 줄씩** 입력 후 엔터 (복사 가능).

```
cd "c:\Users\PC\Desktop\영업 메일링\web"
```

```
npm install
```

```
npm run dev
```

3. **터미널 창은 닫지 말고** 둔 상태에서, 브라우저 주소창에 아래 입력 후 엔터.

```
http://localhost:3000
```

4. 브랜드 목록·**AI 제안 생성**·**메일 발송** 버튼이 보이면 완료.
