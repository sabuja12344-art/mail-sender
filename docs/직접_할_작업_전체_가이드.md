# 1단계: Supabase DB 준비 (brands 테이블 + 확인완료)

## 1-1. Supabase 대시보드 접속

- **어디에서:** 브라우저(Chrome 등)
- **어디로:** 주소창에 아래 입력 후 엔터  
  `https://supabase.com/dashboard`
- **어떤 액션:** 로그인(또는 가입) → 화면에 보이는 **본인 프로젝트** 이름 클릭

---

## 1-2. SQL Editor 열기

- **어디에서:** Supabase 대시보드(왼쪽 메뉴가 보이는 화면)
- **어디로:** 왼쪽 세로 메뉴에서 **SQL Editor** 글자 클릭
- **어떤 액션:** 오른쪽 상단 **+ New query** 버튼 클릭 → 빈 에디터 창이 열리면 OK

---

## 1-3. brands 테이블 생성 SQL 실행

- **어디에서:** 방금 연 SQL Editor 빈 창
- **어디로:** 그대로(에디터 안)
- **어떤 액션:**
  1. 아래 **SQL 전체**를 복사(Ctrl+C)
  2. SQL Editor 빈 칸에 붙여넣기(Ctrl+V)
  3. 오른쪽 아래 **Run** 버튼 클릭
  4. 아래쪽에 에러 없이 결과가 나오면 성공

**▼ 복사할 SQL 전체 (1-3)**

```sql
CREATE TYPE brand_status AS ENUM (
  '수집완료',
  '제안생성',
  '발송대기',
  '발송완료',
  '회신완료'
);

CREATE TABLE public.brands (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  website_url TEXT,
  email TEXT,
  pixel_installed BOOLEAN NOT NULL DEFAULT false,
  analysis_summary TEXT,
  status brand_status NOT NULL DEFAULT '수집완료',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_brands_status ON public.brands (status);
CREATE INDEX idx_brands_created_at ON public.brands (created_at DESC);

ALTER TABLE public.brands ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow authenticated users full access to brands"
  ON public.brands FOR ALL TO authenticated USING (true) WITH CHECK (true);

CREATE POLICY "Allow service role full access to brands"
  ON public.brands FOR ALL TO service_role USING (true) WITH CHECK (true);

COMMENT ON TABLE public.brands IS '수집된 브랜드 정보와 발송 상태 관리';
COMMENT ON COLUMN public.brands.name IS '브랜드명';
COMMENT ON COLUMN public.brands.website_url IS '웹사이트 주소';
COMMENT ON COLUMN public.brands.email IS '추출된 대표 이메일';
COMMENT ON COLUMN public.brands.pixel_installed IS '픽셀 설치 여부';
COMMENT ON COLUMN public.brands.analysis_summary IS 'AI가 분석한 브랜드 약점/특징';
COMMENT ON COLUMN public.brands.status IS '상태';
COMMENT ON COLUMN public.brands.created_at IS '수집 일자';
```

---

## 1-4. 확인완료 상태 추가

- **어디에서:** 같은 SQL Editor 창(방금 Run 한 그 창)
- **어디로:** 에디터 안의 **기존 글자 전체**
- **어떤 액션:**
  1. 에디터 안에서 Ctrl+A(전체 선택) → Delete 또는 백스페이스로 **전부 지우기**
  2. 아래 **한 줄만** 복사해서 에디터에 붙여넣기
  3. **Run** 버튼 클릭
  4. 에러 없으면 성공

**▼ 복사할 SQL 한 줄 (1-4)**

```sql
ALTER TYPE brand_status ADD VALUE '확인완료';
```

- **어떤 액션:** 목록에 **brands** 가 보이는지 확인 → 보이면 1단계 완료

# 2단계: API 키·Secrets 등록 (Gemini, Resend)

## 2-1. Gemini API 키 받기

- **어디에서:** 브라우저
- **어디로:** 주소창에 입력 후 엔터  
  `https://aistudio.google.com/apikey`
- **어떤 액션:** 로그인 → **Create API Key** 버튼 클릭 → 프로젝트 선택(또는 새로 만들기) → 나온 키 **복사** 버튼으로 복사 → 메모장 등에 붙여넣어 두기

---

## 2-2. Resend API 키 받기

- **어디에서:** 브라우저
- **어디로:** 주소창에 입력 후 엔터  
  `https://resend.com`
- **어떤 액션:** **Sign Up** 또는 **Log In** → 로그인 후 왼쪽 메뉴 **API Keys** 클릭 → **Create API Key** 클릭 → Name 아무거나 입력(예: 영업메일링) → **Add** → 나온 키(re_ 로 시작) **복사** → 메모장 등에 붙여넣어 두기

---

## 2-3. Supabase Secrets 화면 열기

- **어디에서:** Supabase 대시보드
- **어디로:** 왼쪽 **맨 아래** **톱니바퀴 아이콘(Project Settings)** 클릭
- **어떤 액션:** 왼쪽에서 **Edge Functions** 탭 클릭 → 화면을 아래로 내려 **Secrets** 섹션 찾기

---

## 2-4. GEMINI_API_KEY 추가

- **어디에서:** Supabase **Settings → Edge Functions → Secrets** 화면
- **어디로:** **Add new secret** 버튼
- **어떤 액션:**
  1. **Add new secret** 클릭
  2. **Name** 칸에 `GEMINI_API_KEY` 입력(또는 복사해 붙여넣기)
  3. **Value** 칸에 2-1에서 복사한 Gemini 키 붙여넣기
  4. **Save** 또는 **Create** 클릭

---

## 2-5. RESEND_API_KEY 추가

- **어디에서:** 같은 Secrets 화면
- **어디로:** 다시 **Add new secret** 버튼
- **어떤 액션:**
  1. **Add new secret** 클릭
  2. **Name** 칸에 `RESEND_API_KEY` 입력
  3. **Value** 칸에 2-2에서 복사한 Resend 키(re_…) 붙여넣기
  4. **Save** 클릭

---

## 2-6. RESEND_FROM_EMAIL 추가

- **어디에서:** 같은 Secrets 화면
- **어디로:** 다시 **Add new secret** 버튼
- **어떤 액션:**
  1. **Add new secret** 클릭
  2. **Name** 칸에 `RESEND_FROM_EMAIL` 입력
  3. **Value** 칸에 `onboarding@resend.dev` 입력(또는 본인 발신 이메일)
  4. **Save** 클릭

# 4단계: 대시보드용 .env.local 만들기

## 4-1. 서비스 롤 키 복사

- **어디에서:** Supabase 대시보드
- **어디로:** 왼쪽 아래 **Settings(톱니바퀴)** 클릭 → 왼쪽에서 **API** 클릭
- **어떤 액션:** **Project API keys** 영역에서 **service_role** 행 찾기 → **Reveal** 버튼 클릭 → 나온 긴 문자열 **전체 선택 후 복사** → 메모장에 붙여넣어 두기

---

## 4-2. web 폴더 열기

- **어디에서:** Windows 탐색기(폴더 창)
- **어디로:** **바탕화면** → **영업 메일링** 폴더 더블클릭 → 그 안 **web** 폴더 더블클릭
- **어떤 액션:** **web** 폴더 안이 보이면 OK(다른 파일들만 보이고 .env.local 은 아직 없음)

---

## 4-3. .env.local 파일 만들기

- **어디에서:** **web** 폴더 안(탐색기)
- **어디로:** 빈 공간에서 **우클릭** → **새로 만들기** → **텍스트 문서** 선택
- **어떤 액션:**
  1. 파일 이름을 **`.env.local`** 로 바꾸기(맨 앞 점 포함, .txt 제거)
  2. 이름 바꿀 때 "확장자를 변경하면 파일을 사용하지 못할 수도 있습니다" 나오면 **예** 클릭
  3. **.env.local** 파일 더블클릭해서 메모장 등으로 열기

---

## 4-4. .env.local 내용 넣기

- **어디에서:** .env.local 을 연 메모장(또는 에디터)
- **어디로:** 빈 칸
- **어떤 액션:**
  1. 아래 **두 줄** 복사해서 붙여넣기
  2. **두 번째 줄**의 `여기_서비스_롤_키_붙여넣기` 를 **4-1에서 복사한 서비스 롤 키 전체**로 지우고 붙여넣기
  3. **파일** → **저장** 후 닫기

**▼ 복사할 두 줄 (4-4)**

```
SUPABASE_URL=https://rchflauwrrlfefxzbtks.supabase.co
SUPABASE_SERVICE_ROLE_KEY=여기_서비스_롤_키_붙여넣기
```
